# Задание 1. 

# Master-Slave (ведущий-ведомый)

* Особенности:

1) Один master (основной) сервер принимает все операции записи

2) Один или несколько slave (ведомых) серверов только читают данные

3) Репликация происходит однонаправленно: от master к slave

4) Изменения на slave не передаются обратно на master

* Преимущества:

1) Простота настройки и обслуживания

2) Отсутствие конфликтов при записи (только один источник)

3) Возможность распределения нагрузки чтения между slave

4) Надежное резервное копирование (можно делать дамп с slave)

* Недостатки:

1) При отказе master требуется ручное переключение на slave

2) Задержки репликации могут влиять на актуальность данных

# Master-Master (ведущий-ведущий)

* Особенности:

1) Оба сервера могут принимать операции записи

2) Двунаправленная репликация между серверами

3) Каждый сервер является одновременно и master, и slave

* Преимущества:

1) Высокая доступность (при отказе одного сервера второй продолжает работу)

2) Возможность распределения нагрузки записи

3) Географическое распределение (запись в ближайший датацентр)

* Недостатки:

1) Сложность разрешения конфликтов при одновременной записи

2) Риск "зацикливания" репликации

3) Необходимость использования auto_increment_increment и auto_increment_offset
